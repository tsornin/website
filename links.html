Game Engine
Managing Game States in C++ « Game Dev Geek
A nice design for a game engine with states. The implementation given has a bug though: CGameEngine:: HandleEvents, Update, and Draw are not guarded by a ( !states.empty() ) check. CGameState::Update might use CGameEngine::PopState to empty the stack (very possible since it could be the last state in the stack). Even if the state knows it's going to be cleaned up and doesn't do anything else, CGameEngine::Update will finish, and then CGameEngine::Draw will look for a state on the stack and crash. One solution would be to add the checks; another would be queueing up state actions and performing them at the end of each game loop.
